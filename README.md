# SNHU_CS_Portfolio
Portfolio showcasing software engineering projects developed during my Computer Science program at SNHU.


The Corner Grocer Tracking program was built to help a local store analyze their daily produce sales. They needed a reliable way to process a text file full of purchased items and easily see exactly how many of each item were sold. I think I did a particularly good job implementing a map data structure to handle this problem. Maps are perfect for pairing a grocery item's name directly with its purchase count. I was also pretty happy with the input validation on the user menu. It catches invalid keyboard entries and prevents the program from crashing if someone accidentally types a letter instead of a number.

There is definitely room for improvement if I were to build on this further. I could enhance the code by adding better error handling for the file input process. If the daily text file is missing or corrupted, the program needs a much safer way to alert the user instead of just failing. Taking the time to build out those safeguards would make the software much more secure and reliable in a real business environment. Getting the file input and output working smoothly was actually the most challenging part of this project for me. Reading the input list and simultaneously writing the backup frequency.dat file took some trial and error. I overcame those roadblocks by taking things one step at a time and leaning heavily on official C++ documentation to make sure my syntax was correct.

The skills I practiced here are going to be incredibly transferable to future coursework and personal projects. Dealing with file structures and organizing data with maps are foundational concepts that pop up constantly in software engineering. To make sure this specific program stays maintainable and adaptable, I kept the code organized and straightforward. I used clear variable names and separated the logic into distinct sections. The background file reading happens immediately on startup, and the user interface logic is kept separate in the menu loop. That design choice makes it much easier for someone else to read through the logic and update the code later on without breaking the core functionality.
