# SNHU_CS_Portfolio
The Gaming Room wanted to take their physical game, Draw It or Lose It, and turn it into a web-based application. They needed a software design capable of handling a shared ecosystem where multiple teams and players could interact simultaneously across different devices. I think I did a particularly great job breaking down the memory management and network constraints in the documentation. Figuring out how to prevent the central server from crashing under a heavy load by passing string references instead of raw image data was a fun challenge. I also put a lot of effort into evaluating the server architectures to justify a highly scalable and cost-effective Linux setup.

Working through the design document first was incredibly helpful when it came to visualizing the actual code. Defining the UML domain model for the Entity, Game, Team, and Player classes gave me a solid blueprint, so I knew exactly what attributes each class needed before writing any logic. If I could revise one part of this work, I'd definitely expand on the visual diagrams by adding a network topology map or sequence diagrams to make the API communication flow easier to understand at a glance. I interpreted the user's needs by focusing heavily on performance and accessibility. Players just want a fast and lag-free experience whether they're on a phone or a desktop. That's exactly why I recommended lazy loading and keeping the network payloads lightweight. It's crucial to consider the end user because if the app feels clunky, people simply won't play it.

My approach to designing this software was to tackle the biggest bottlenecks first. I focused heavily on how constraints like memory and latency would impact the overall architecture. In the future, I'll definitely carry forward this constraint-first strategy. It actually reminds me a lot of how I structure the agentic pipelines and isolated components in my own personal projects, like KalClaw. Breaking a large application down by its core limitations makes building robust systems a whole lot easier.
